# Маппинг корзины Coffee Stop Soft -> `positions` в MobCashier

## Цель
Определить, как превращать внутренние линии заказа (`order_lines` + модификаторы) в формат `positions` для `POST /api/v1/orders/create` в MobCashier.

## Ограничение MVP (без усложнения кофейни)
В “кофейне” мы используем ваш существующий модельный слой:
- меню/категории/позиции
- модификаторы
- итоговая цена за линию

В Эвотор не добавляем новые “evotor-specific” поля на уровне меню. Мы просто отправляем позиции, которые соответствуют уже сформированным строкам заказа.

## Общий принцип
Каждая строка заказа = одна `position` в Evotor.

## Для каждой `position` передаём
Используем значения из вашего backend расчёта:

1. `name`
   - базово: `menu_item_name_snapshot`
   - при наличии модификаторов добавляем текстом:
     - например: `Латте (Молоко: Овсяное, Сироп: Ваниль)`

2. `quantity`
   - `order_lines.quantity`

3. `price`
   - `order_lines.unit_price_cents / 100` (в рублях)
   - модификаторы уже учтены в вашей `unit_price_cents` (как сейчас)

4. `settlement_method_type`
   - `FULL`

5. `type`
   - `NORMAL`

6. `tax`
   - `NO_VAT` (MVP зафиксировано)

## Контакты
- `client_email` = email гостя
- телефон не передаём (email-only MVP)

## Идемпотентность
Рекомендуется хранить в БД:
- `order_id -> receipt_uuid`
- факт обработки callback по `receipt_uuid`

Чтобы исключить проблемы с повторной фискализацией одного и того же чека.
