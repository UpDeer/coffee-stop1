# Стек и цель продукта

## Бэкенд

- **FastAPI** + **Uvicorn** — HTTP API по [api-v1.md](api-v1.md).
- **PostgreSQL** — данные; схема в [../../backend/schema.sql](../../backend/schema.sql); миграции планируются через **Alembic** (после стабилизации моделей).
- **SQLAlchemy 2.0** (sync) — доступ к БД; при необходимости позже можно перейти на async.

## Цель: полноценный рабочий сервис (не MVP)

Разрабатываем **production-ready** приложение:

- Конфигурация через переменные окружения (секреты не в репозитории).
- Разделение **health** (процесс жив) и **readiness** (БД доступна).
- Структурированное логирование, единый формат ошибок API.
- Идемпотентность платежей, аудит событий заказа (`order_events`).
- Доставка заданий на печать: **polling** агентом (см. [stage0-decisions.md](stage0-decisions.md)).

Дополнительно по мере готовности: rate limiting, метрики, тесты, CI.
