# Сценарии и экраны: заказ по QR и выдача

Этап 0 — проектирование. Два контура: **гость** (веб по QR) и **бариста** (админка).

---

## 1. Статусы заказа (справочно)

| Статус | Кто меняет | Смысл |
|--------|------------|--------|
| `draft` | гость / система | Корзина, заказ ещё не оплачен |
| `payment_pending` | система | Ожидание подтверждения оплаты |
| `paid` | система (webhook) | Оплата прошла; заказ сразу считается «в процессе готовки», попадает в очередь бариста |
| `ready` | бариста | Готов к выдаче; автоматическая печать чека с номером + уведомление гостю |
| `picked_up` | бариста / таймаут | Забрали (опционально) |
| `cancelled` | бариста / гость | Отменён до готовности |
| `refunded` | система / бариста | Возврат после оплаты |

## 2. Сценарий: гость (заказ по QR)

### 2.1. Пользовательские истории

- **Как гость**, я прихожу в точку, вижу QR-код, сканирую — и сразу открывается страница составления заказа (меню этой точки).
- **Как гость**, я вижу меню (напитки, еда), могу добавить позицию в корзину и настроить её (молоко, сироп, размер и т.д.).
- **Как гость**, я вижу корзину, итог и кнопку «Оплатить»; после оплаты получаю подтверждение и номер заказа.
- **Как гость**, я вижу статус заказа («Готовится» / «Готово, можно забирать») и при готовности получаю уведомление (Web Push или email).
- **Как гость**, я забираю заказ на столе выдачи по номеру на чеке под стаканом.

### 2.2. Экраны (гость)

**Открытие по QR (сразу меню)**

- URL вида: `https://<домен>/s/<storeSlug>?t=<token>`. По скану QR гость попадает **сразу на страницу составления заказа** — меню точки, без промежуточного экрана «ландинга». Название точки можно показывать в шапке.

**Меню**

- Список категорий и позиций (напитки, еда).
- Желательно: **фотографии** напитков и блюд в карточках.
- Карточка позиции: фото (если есть), название, цена, кнопка «В корзину». Переход в кастомизацию (если есть опции) или сразу добавление в корзину.
- Иконка корзины с счётчиком в шапке.

**Кастомизация позиции (модалка или отдельный экран)**

- Выбор размера, типа молока, сиропа, добавок (по наличию в точке). Итоговая цена позиции. Кнопка «Добавить в корзину».

**Корзина**

- Список позиций с составом и ценой, возможность изменить количество или удалить.
- Итоговая сумма. Поле для email (опционально, для чека и уведомления о готовности). Кнопка «Оплатить».

**Оплата**

- Редирект на страницу платёжного провайдера или виджет оплаты. После успеха — редирект обратно на экран «Заказ принят».

**Заказ принят / Статус заказа**

- Крупно: номер заказа (например, 042).
- Текст: «Заказ принят, ожидайте приготовления» → «Готовится» → «Готово, можно забирать. Номер заказа 042».
- Обновление статуса: polling или SSE. При готовности — запрос разрешения на Web Push (если ещё не дано) и показ уведомления.
- Опционально: **анимация или мини-игра** в ожидании (по аналогии с игрой в динозавра в Chrome при ожидании загрузки), чтобы скрасить ожидание.

**Ошибки**

- «Оплата не прошла» — **указывать причину** (например: «Карта отклонена», «Недостаточно средств», «Таймаут оплаты» — по ответу провайдера). Кнопки: «Попробовать снова» / «Вернуться в корзину».
- «Точка временно не принимает заказы» — показать при недоступности или отключении приёма.

## 3. Сценарий: бариста (админка)

### 3.1. Пользовательские истории

- **Как бариста**, я вижу очередь заказов в двух колонках: «Новые» (`paid`) и «Готово» (`ready`). Весь состав заказа гостя виден сразу в карточке, без раскрытия.
- **Как бариста**, я переношу заказ в «Готово» одним действием; при этом автоматически печатается чек с номером заказа и гостю уходит уведомление о готовности.
- **Как бариста**, я могу отменить заказ или инициировать возврат (refund) при необходимости.

### 3.2. Экраны (бариста)

**Вход**

- Логин/пароль (или другой способ авторизации для сотрудников точки). Выбор точки, если у пользователя несколько.

**Очередь заказов**

- Две колонки: **«Новые»** (`paid`) и **«Готово»** (`ready`).
- В колонке «Новые» каждая карточка сразу показывает **весь заказ целиком**: номер заказа (крупно), время, сумма, полный состав (позиции, модификаторы, комментарии). Раскрытие по клику не требуется — всё видно в списке.
- В колонке «Готово» — заказы, ожидающие выдачи; тот же формат карточки.

**Действия с заказом**

- Кнопки на карточке в «Новые»: **«Готово»** (перевод paid → ready), **«Отменить»**.
- При нажатии «Готово»: заказ переносится в колонку «Готово», **автоматически** создаётся задание на печать чека с номером заказа (без дополнительных действий бариста) и отправляется уведомление гостю. Ручной перенос иконок между колонками не нужен.
- Чек по оплате отправляется гостю на email (если указан); для точки печатается только **чек с номером заказа** при переводе в «Готово».

**Печать**

- Печать при переводе в `ready` — **автоматическая** (без отдельного нажатия «Печать»). Кнопка «Печать ещё раз» только на случай сбоя принтера (для заказов уже в «Готово»).

**Настройки точки (минимально)**

- Включение/выключение приёма заказов для точки (опционально для MVP).
