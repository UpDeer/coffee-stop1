# ИБ-чеклист: Tochka API + Evotor фискализация

## 0. Модель угроз (кратко)
Нужно защитить:
- целостность статуса оплаты (чтобы `paid` не подменяли злоумышленники)
- целостность фискализации (чтобы нельзя было повторно/неверно фискализировать чеки)
- секреты (токены Tochka/Evotor, ключи вебхуков)
- персональные данные (email/телефон — по минимуму)

## 1. Tochka webhook безопасности
1) Проверять подлинность webhook:
   - body webhook — JWT (RS256)
   - проверять подпись публичным ключом Tochka (OpenAPI public key)
2) Проверять `webhookType == acquiringInternetPayment`
3) Проверять подписи/TTL:
   - rejecting if token invalid/expired
4) Идемпотентность (защита от replay):
   - дедуп по `operationId`
   - второй webhook с тем же `operationId` не меняет статусы и не создаёт повторные события
5) Ограничение входа:
   - webhook endpoint принимает только `POST`
   - ограничить размер тела и тип контента
   - rate limit / WAF (если есть перед backend)

## 2. Evotor безопасность
1) Секреты Evotor:
   - login/password/group_code хранить только в env/secret manager
2) HTTPS:
   - все обращения к Evotor делаются сервер-сервер по HTTPS
3) Идемпотентность фискализации:
   - дедуп/lock на уровне `order_id` и/или `fiscal_uuid` (который вернёт Evotor)
4) Логирование:
   - запрещено логировать логины/пароли и секреты токенов
   - в `raw_payload` хранить только необходимое, маскировать PII при логировании

## 3. Общие требования для production
1) Secrets:
   - не коммитить env/ключи
   - маскирование чувствительных полей в логах
2) Валидация входных данных:
   - сумма/кол-во/цены только серверно
   - проверять согласованность суммы `order.total_cents` перед фискализацией
3) Ошибки и ретраи:
   - backoff для временных ошибок (5xx, network timeouts)
   - после N попыток — перевод `failed` + алерт в логах/админке
4) Наблюдаемость:
   - correlation id: `order_id` + `operationId`
   - метрики: webhook error rate, fiscalization success rate, latency
5) Контроль доступов:
   - для бариста-админки в production нужен отдельный слой auth
   - (если пока нет) хотя бы IP allowlist / базовая авторизация
